Thank you for your interest in contributing to rspamd-quarantine!
Contributions of all kinds are welcome: bug reports, feature requests,
documentation, and code.

## How to Contribute

### Reporting Bugs
- Use the issue tracker
- Include steps to reproduce the problem
- Include logs or error messages where possible
- Specify your environment (OS, Rspamd version, etc.)

### Suggesting Enhancements
- Describe the use case clearly
- Explain why the change would be useful
- Be open to discussion and alternative approaches

### Code Contributions

1. Fork the repository
2. Create a feature or fix branch
3. Make your changes with clear, concise commits
4. Ensure existing functionality is not broken
5. Submit a pull request with a clear description

### Coding Guidelines
- Follow existing code style and conventions
- Keep changes focused and minimal
- Add comments where behavior is non-obvious
