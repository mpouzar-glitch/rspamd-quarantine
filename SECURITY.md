## Reporting a Vulnerability

If you believe you have found a security vulnerability, **please do not open a public issue**.

### How to report
- Send a detailed report to: **<pouzar@milnet.cz>**
- Include:
  - A clear description of the issue
  - Steps to reproduce (if possible)
  - Potential impact
  - Affected versions
  - Any relevant logs or proof-of-concept code

### What to expect
- You should receive an acknowledgment **within 72 hours**
- We will investigate and assess the report promptly
- If the vulnerability is confirmed:
  - A fix will be developed for supported versions
  - A security release may be issued
  - Credit will be given to the reporter if desired
- If the report is declined, we will explain the reasoning where possible

### Coordinated disclosure
We ask reporters to allow reasonable time for remediation before any public disclosure.
Responsible disclosure helps protect users and maintainers alike.

Thank you for helping keep rspamd-quarantine secure.
